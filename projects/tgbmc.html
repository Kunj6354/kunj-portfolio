TGBMC Tactical GIS Platform

A modern geospatial platform for situational awareness, counter-drone monitoring, mission planning, replay, sensor visualization, and spatial decision support.

Project overview

TGBMC is a tactical GIS and battle-management software proof of concept designed around geospatial awareness, planning, replay, analytics, and sensor information.

The project is positioned as a professional GIS and decision-support platform rather than an autonomous engagement or weapon-control system.

My contribution

  • Defined the implementation stack and overall application architecture.
  • Designed the PostgreSQL and PostGIS data foundation for geospatial entities and application state.
  • Organized Angular modules and navigation around clear operational workflows.
  • Planned Luciad RIA integration for professional 2D and 3D GIS visualization.
  • Established FastAPI, Redis, WebSocket, and Docker foundations for backend services and live updates.
  • Reviewed visible terminology to keep the platform focused on awareness, planning, replay, and geospatial intelligence.
  • Developed the platform incrementally, one module and milestone at a time.

Core platform areas

Situational awareness

Present sensor information, geospatial entities, operational zones, sites, tracks, and events through one map-centred interface.

Mission planning

Support planning workflows using map geometry, spatial context, stored plans, overlays, and relevant operational information.

Replay and analysis

Preserve time-based records so scenarios and events can be reviewed for training, investigation, and system assessment.

Geospatial intelligence

Use PostGIS-backed spatial data and professional GIS visualization for querying, filtering, and understanding location-based information.

Sensor dashboards

Organize system health, detections, alerts, and sensor state into clear dashboard and map workflows.

Engineering decisions

GIS-first product design

The map is treated as the primary workspace rather than being added as a secondary visualization.

Clear domain boundaries

Backend services, persistent spatial data, live runtime state, and frontend visualization are separated to keep the platform easier to test and extend.

Controlled terminology

Public and user-facing language emphasizes monitoring, awareness, planning, replay, and decision support.

One-module-at-a-time delivery

The project avoids changing every area simultaneously. Each module is inspected, implemented, validated, and then integrated.

Outcome

TGBMC now has a defined product direction, modern application architecture, clean geospatial database foundation, professional terminology, and a structured path for implementing GIS modules incrementally.

This public case study covers only software architecture, geospatial workflows, platform design, and decision-support concepts. Sensitive operational implementation details are not included.
