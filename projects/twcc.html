TWCC Legacy Recovery & Modernization

Recovery of a production-era, multi-module C++ and Qt GIS system, including desktop displays, planning tools, simulation, replay, database integration, network communication, and a modern web POC.

Project context

The system was developed over several years as a collection of connected C++ and Qt applications. Recovering it required more than making the code compile: its original workflows, GIS behaviour, runtime dependencies, database assumptions, and communication between modules also had to be understood and validated.

My approach treated the recovered user interfaces, C++ backend, database tables, network behaviour, and historical runtime evidence as the source of truth.

My contribution

  • Inspected and rebuilt unfamiliar legacy C++ and Qt modules.
  • Recovered GIS displays, overlays, track interaction, planning, simulation, and replay workflows.
  • Traced UDP/TCP communication and PostgreSQL-backed runtime behaviour.
  • Corrected geospatial handling while preserving source-backed workflow behaviour.
  • Made machine-sensitive runtime behaviour configurable where practical.
  • Built a modern Angular, Django, and Luciad-based interpretation as a separate POC.
  • Validated changes through focused builds, runtime checks, database checks, and end-to-end workflow testing.

Primary engineering challenges

Recovering behaviour without inventing it

Missing or incomplete workflows were restored from code, database, interface, and runtime evidence rather than redesigned from assumptions.

Distributed assumptions on one development machine

The original system expected multiple connected applications and machines. A reliable development environment required careful port, address, startup, and module coordination.

Legacy GIS and coordinate behaviour

Geospatial operations had to preserve expected map behaviour, coordinate conventions, overlays, visibility, and interaction.

Controlled modernization

The web POC demonstrates a modern architecture without presenting newly simulated behaviour as original operational truth.

Validation approach

Build verification Rebuilt affected C++ and Qt applications after focused changes.
Runtime verification Checked startup, connectivity, logs, module behaviour, and error conditions.
Data verification Validated relevant database records and runtime state changes.
Workflow verification Exercised restored operator actions from input through visible result.

Outcome

The work established a credible recovery path for the original desktop system and its supporting applications while also showing how selected concepts could be represented through a modern Angular, Django, and GIS architecture.

This public case study describes the engineering process at a high level. Sensitive operational logic, internal protocols, deployment topology, credentials, and restricted implementation details are intentionally excluded.
